# 📱 阶段三详细设计：手机 App 端（Terminal UI）技术方案

## 1. 技术选型与核心组件

* **开发框架**：**Flutter**（Dart 语言）。一套代码完美编译出 iOS 和 Android 两个原生客户端，极其适合独立开发。
* **网络隧道内核**：引入 **`flutter_wireguard`** 或包装开源的 Tailscale 手机端核心组件（`libtailscale`）。当用户登录 App 后，App 内部会自动拉起手机本地的 VPN 网卡，连入你的 Headscale 虚拟网。
* **终端渲染器（核心）**：使用 **`xterm` (Flutter 包)**。这是一个在 Flutter 纯原生渲染的、兼容 `Xterm.js` 协议的精美终端组件，支持主流 Linux 颜色高亮、按键修饰符、以及流式文本回显。

## 2. 手机 App 核心运行逻辑与页面状态机

App 的页面层级极其精简，一共只需要 3 个核心页面：

```text
[登录页 (NCLoginScreen)] 
       │ (国内手机号验证码 / 微信登录)
       ▼
[设备大厅 (NCDeviceListScreen)] ────(点击右上角扫码)────► [扫码绑定页 (NCScannerScreen)]
       │                                                          │
       │ (点击已在线的 Linux 设备)                                  │ (扫描 Linux 终端二维码)
       ▼                                                          ▼
[极客终端面板 (NCTerminalScreen)] ◄─────────────────────────(云端自动授信网络打通)

```

### A. 核心交互流程细节

#### ① 登录与网卡激活 (NCLoginScreen)

* 用户输入手机号登录，App 向云端发送请求，获取该用户的 JWT Token 以及专属的 `Namespace`。
* **关键底层操作**：App 内部启动本地的隧道内核，向你的控制面（`api.nextconnect.com`）发起握手。手机的通知栏会弹出系统的 **VPN 图标**。此时，手机正式分配到了一个形如 `100.64.0.2` 的内网虚拟 IP。

#### ② 扫码绑定（NCScannerScreen）

* 用户点击“添加设备”，唤起手机摄像头，扫描 Linux 终端打印出来的二维码。
* 解析二维码中的 URL，提取出 `pairing_token`。
* 手机端立刻携带自己的 JWT 身份令牌，向云端发送：`POST /api/v1/pair/confirm`，附带 `{"pairing_token": "NC-789231"}`。
* 云端收到后完成授信，用户刷新设备大厅，刚才那台 Linux 电脑（例如 IP 为 `100.64.0.3`）就会亮起绿灯，显示“在线”。

#### ③ P2P 终端连接 (NCTerminalScreen)

* 用户在设备列表里点击这台 Linux 电脑。
* **绝妙的纯本地连接**：App 内部直接实例化一个标准的 Dart SSH 客户端（使用 `dartssh2` 库）。它不去连接任何外网服务器，而是**直接连接 `100.64.0.3:22**`！
* 此时，流量完全通过本地的 VPN 网卡，在 5G 基站/Wi-Fi 与你家里的电脑之间进行 **P2P 硬件级打洞直连**。
* 连接成功后，将 SSH 的数据流（Stdin/Stdout）直接绑定到 `xterm` 视图组件上。
