# 🚀 项目总纲：NextConnect (极简极客 P2P 组网终端工具)

## 📌 1. 项目核心定位与设计哲学

* **核心痛点**：解决国内 AI 开发者、高校研究生无脑登录、免翻墙、低延迟远程控制本地/公司/实验室 Linux 终端（WSL2/服务器）的刚需。
* **设计哲学**：**极简主义。** 砍掉一切 WebUI 映射、文件传输等臃肿功能，**只做一件事：在手机端高颜值、顺畅地通过 SSH 敲命令行**。
* **技术路线**：基于 **Headscale (控制面)** + **Tailscale Core (数据面)**。主打 P2P 直连（利用校园网 IPv6 或普通 NAT 穿透），平台方（我方）服务器**只跑控制信号，百分之百不中转用户流量**。

## 🏗️ 2. 全盘系统架构拓扑

```
                              │   - 用户数据库 / 微信微信扫码登录认证   │
                              │   - Headscale 控制面进程 (负责设备握手)│
                              └───────────────────┬────────────────────┘
                                                  │
                                   (仅在设备启动/配对时进行控制面交互)
                                                  │
                   ┌──────────────────────────────┴──────────────────────────────┐
                   ▼                                                             ▼
┌──────────────────────────────────────┐                       ┌──────────────────────────────────────┐
│       用户 Linux 电脑端 (WSL2)        │◄─────────────────────►│            用户手机 App 端           │

│ - 封装开源 Tailscale 核心组件          │  [加密 WireGuard 隧道] │ - 内置 Xterm.js 渲染的终端面板       │

```

## 🛠️ 3. 核心开发里程碑与模块划分

我将整个开发计划切分为 **四大核心阶段**，后续我们将针对每一阶段向 Claude Code 下达绝对具体的局部指令：

### 🧱 阶段一：云端控制中心（Control Plane）搭建与改造

* **核心任务**：基于 Go 语言，将开源的 **Headscale** 封装为我们的云端中央指挥官。
* **关键开发项**：
* 移除 Headscale 原生的 OIDC/命令行注册设备逻辑。
* 开发一套面向国内的 **Bridge API（桥接网关）**，支持国内手机号验证码/微信扫码登录，自动为用户划分专属的虚拟内网（Namespace）。
* 提供设备配对接口：接收 Linux 端传来的 Token，在云端自动完成设备授信（Device Approve）。



### 💻 阶段二：Linux 客户端（Daemon）傻瓜式安装包

* **核心任务**：编写一个无脑的 `install.sh` 脚本和后台守护进程，让用户复制粘贴即可完成内网打洞。
* **关键开发项**：
* 自动化环境检测（检测是否为 WSL2、Systemd 状态、SSH 22端口是否开启）。
* 下载并精简开源的 Tailscale 二进制内核，剥离无关组件。
* 硬编码修改（或启动参数注入）统一指向我们的云端控制域名：`tailscale up --login-server=https://api.nextconnect.com`。
* 在终端打印出唯一的配对二维码（包含设备 UUID 和云端绑定 Token）。



### 📱 阶段三：手机 App 端（Terminal UI）开发

* **核心任务**：使用 **Flutter**（一套代码跨 iOS/Android）开发一款高颜值的极客移动终端。
* **关键开发项**：
* 极简登录页：国内手机号/微信扫码登录，获取云端 Namespace 令牌。
* 扫码绑定功能：扫描 Linux 终端生成的二维码，调用云端 API 完成 P2P 网络闭环。
* **核心组件集成**：集成开源的 **`Xterm.js`**（或 Flutter 对应的终端渲染库），顺着本地分配的 `100.X.X.X` 的内网 IP，直接唤起纯本地 SSH 连接，提供比 Termius 更流畅的打字回显。



### 🚨 阶段四：网络边界安全、免责防御与产品上线

* **核心任务**：建立法律合规与安全边界，防止产品被滥用导致开发者“踩缝纫机”。
* **关键开发项**：
* **流量熔断机制**：在云端控制器硬编码限制，该虚拟网络内**仅允许放行 22 端口（SSH 流量）**，从物理上掐断用户通过本工具映射内网非法网站（80/443）的可能。
* 用户实名认证流引入，确保所有内网穿透行为可追溯（满足国内网安合规要求）。
* 产品打包、私有化部署说明（支持硬核极客自备 DERP 中转服务器）的文档编写。
